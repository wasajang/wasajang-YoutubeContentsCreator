## 1. IdeaPage - 씬 개수 직관성

- 씬 10개 선택이 기본인데, 씬 선택의 의미가 직관적이지 않음
- **씬 개수 = 생성할 시작 이미지 개수**라는 것을 표시
- Generate Script 하단에 씬 개수에 따른 시작 이미지 개수가 시각적 UI로 구성

---

## 2. IdeaPage - 레이아웃 개편 (2분할 → 3분할)

### 기존 문제
- 나레이션 프리셋/아트스타일/화면비율이 우측 사이드바 느낌으로 빠져있음
- IdeaPage가 "대본 + 설정 선택" 2가지를 하는 단계라는 걸 유저가 놓침

### CEO 최종 결정: 3분할 레이아웃
```
┌─────────────────┬──────────────────┬─────────────────┐
│  1. 대본 작성    │  2. 설정 선택     │  3. 결과        │
│                 │                  │                 │
│ - 대본 입력     │ - 프리셋 선택     │ - Script        │
│ - 아이디어 입력  │ - 아트 스타일     │   Breakdown     │
│ - 씬 개수 선택  │ - 화면 비율       │ - 씬 목록       │
│                 │                  │ - 편집/체크      │
│ [하단: 씬 개수  │                  │                 │
│  = 이미지 N장]  │                  │                 │
└─────────────────┴──────────────────┴─────────────────┘
```

### 단계별 진행 UX
- 3단계이므로 유저가 단계별로 직관적으로 진행하도록
- 각 단계 완료 시 다음 단계 버튼 활성화
- 진행 안내 UI 필요

---

## 3. 프리셋 선택 시 AI 대본 연동

### 프리셋/템플릿 선택 시
- 선택하면 "이 프리셋의 대본 작성 규칙에 따라 AI가 대본을 생성합니다" 안내
- 아이디어 입력 → AI 대본 생성 시 프리셋의 scriptSystemPrompt가 적용됨을 사용자에게 알림

### 대본 직접 입력 시 (프리셋 선택된 상태에서)
- 팝업: "이미 설정된 대본 작성 규칙에 따라 N씬을 추천합니다. 대본을 수정할까요?"
- **예**: 프리셋 규칙에 맞게 씬 개수 추천 + 대본 수정 진행
- **아니요**: 수동 씬 개수 설정 + 현재 대본 유지 → Script Breakdown 진행

### 프리셋 선택 확인 팝업
- 프리셋 클릭 시 "이 템플릿의 설정과 프롬프트를 모두 적용할까요?" 팝업
- 예 → 적용 / 아니요 → 프리셋 선택 안 됨

## 4. StoryboardPage - AI 대본 분석 팝업 개선

### 4-A: 템플릿 선택한 유저
- 팝업 내용: "선택한 템플릿 '{이름}'에 저장된 캐릭터({해골1, 해골2}) 등을 자동 추천합니다"
- "예 AI로 분석하기" 클릭 시:
  - 템플릿 castPreset에 정의된 카드 수에 맞게 추천
  - 유저 캐스트에서 일치하는 카드 찾기
  - 부족하면 "추가 카드 생성" 팝업 → 생성 후 선택 완료

### 4-B: 템플릿 미선택 유저
- 팝업 내용: "템플릿을 선택하지 않았으므로, 대본 분석을 통해 카드 추천/생성을 진행합니다"
- "예 AI로 분석하기" 클릭 시:
  - 대본 분석 → 캐릭터/배경/아이템 수량 결정
  - **프롬프트는 추후 dev팀이 입력 (현재는 mock 데이터)**
  - **TODO 표시 필요: "실제 AI 프롬프트 추후 입력 필요"**
- "아니오 기본 카드 사용" 클릭 시:
  - 최대 5개 선택 가능
  - 새 카드가 필요하면 카드 생성 가능

## 4-2. StoryboardPage 컷 분할 레이아웃

- 1단계에서 선택한 카드 덱이 좌측 사이드바로 표시
- 분할된 컷 + 영상 개수 선택이 우측에 양분할
- **일괄 영상 개수 선택 버튼** 추가 (한번에 모든 컷에 2 또는 3 적용)

---

## 5. 컷 분할 - 이미지 박스 시각화

- 영상 개수 선택에 따라 **우측에 이미지 박스** 표시
  - 1 선택 → 이미지 박스 1개
  - 2 선택 → 이미지 박스 2개 (2-1, 2-2)
  - 3 선택 → 이미지 박스 3개 (3-1, 3-2, 3-3)
- 컨테이너 박스 높이에 맞게 분리

---

## 6. Generate 페이지 분리

### CEO 제안: /project/storyboard → /project/generate 분리

- `/project/storyboard`: 카드 선택 + 컷 분할
- `/project/generate`: 이미지/영상 생성 (새 페이지)

### 6-A: 템플릿 선택 유저
- 각 분할된 씬별로 이미지/영상 생성 프롬프트가 준비됨
- 템플릿의 promptRules 활용
- 이미지 프롬프트 박스: "이미지 생성 클릭하면 생성됩니다"
- 영상 프롬프트 박스: "영상 생성 진행시 프롬프트 제작됩니다"

### 6-B: 템플릿 미선택 유저
- "추가로 반영하고 싶은 내용이 있나요?" 질문
- 있으면 → 내용 입력받아 프롬프트에 반영
- 없으면 → 화면비율 + 아트스타일 + 씨드카드 + 대본을 조건으로 AI 호출하여 프롬프트 자동 생성
