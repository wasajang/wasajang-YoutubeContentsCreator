## 섹션 1: Vrew 개요

### 1.1 Vrew란?

Vrew(브루)는 한국 스타트업 **VoyagerX**가 2017년 3월에 개발한 AI 기반 영상 편집 프로그램이다.
핵심 슬로건: **"문서 편집하듯 쉬운 영상 편집"**

- **최신 버전**: 3.5.4 (2026년 2월 기준)
- **플랫폼**: Windows, macOS, Ubuntu (데스크톱) / Android, iOS (모바일, 2026년 7월 종료 예정)
- **개발사**: VoyagerX Inc. (한국)
- **공식 사이트**: https://vrew.ai

### 1.2 타겟 유저

| 유저 타입 | 사용 목적 |
|-----------|----------|
| YouTube 크리에이터 | 자막 자동 생성, TTS 나레이션 영상 제작 |
| 교육자/강사 | 온라인 강의 영상 자막 삽입 |
| 기업 마케터 | 제품 소개 영상, 홍보 콘텐츠 |
| 팟캐스트 운영자 | 오디오 → 영상 변환 |
| 영상 편집 초보자 | 전문 편집 툴 없이 간단한 영상 제작 |

### 1.3 핵심 차별화

기존 영상 편집기(Premiere, KineMaster)와 달리 **타임라인이 아닌 텍스트(문서)가 편집의 중심**이다.
- 자막 텍스트를 수정하면 → 영상도 함께 편집됨
- 텍스트를 삭제하면 → 해당 구간의 영상도 삭제됨
- 텍스트 순서를 바꾸면 → 영상 순서도 변경됨

## 섹션 2: 핵심 워크플로우

### 2.1 프로젝트 시작 방식 (4가지 진입점)

Vrew는 프로젝트 생성 시 4가지 시작 방식을 제공한다:

```
1. 새 영상 파일로 시작하기  → 기존 영상 업로드 → 음성 인식 → 자막 생성
2. 텍스트로 비디오 만들기   → 대본 입력 → AI 음성(TTS) + AI 이미지 자동 생성
3. 사진으로 동영상 만들기   → 이미지 업로드 → 자동 애니메이션 + 자막/TTS 추가
4. 영상 녹화로 시작하기     → 화면 녹화 → 자동 자막 생성
```

### 2.2 TTS 기반 영상 제작 워크플로우 (핵심 플로우)

이것이 AntiGravity에 가장 관련이 깊은 워크플로우다.

```
[1단계: 대본 입력]
    ├── 직접 입력 (텍스트 붙여넣기)
    ├── AI 대본 생성 (주제만 입력하면 AI가 스크립트 작성)
    └── 파일 불러오기 (.txt 등)
         │
         ▼
[2단계: AI 음성(TTS) 선택]
    ├── 5개 국어 (한/영/일/중/스페인어)
    ├── 200+ AI 음성 (남/녀/어린이/어르신 등)
    ├── 속도 조절 (0.9~1.05 자연스러운 범위)
    ├── 피치 조절 (±1 반음)
    └── 내 목소리 AI 클론 기능
         │
         ▼
[3단계: 자동 클립 분할 + 타이밍 동기화]
    ├── AI가 대본을 문장/구간 단위로 자동 분할
    ├── 각 클립 = "영상줄(위)" + "자막줄(아래)" 2층 구조
    ├── TTS 음성 길이에 맞춰 각 클립 타이밍 자동 설정
    └── 글자 수 기준으로 클립 나누기 가능 (AI 자동 클립 나누기)
         │
         ▼
[4단계: 이미지/영상 매칭]
    ├── AI 이미지 자동 삽입 (핵심 기능!)
    │   ├── [삽입] → [AI 이미지 자동삽입] → [전체클립] → [삽입]
    │   ├── 한 번에 최대 10개 클립씩 처리
    │   ├── 15가지 스타일 중 선택 (사실적 사진 ~ 추상적 아트)
    │   ├── "이미지 프롬프트" 기능으로 전체 분위기 설정
    │   └── 무료 비디오 포함 옵션 → 일부는 스톡 이미지/영상으로 자동 채움
    ├── 무료 에셋 라이브러리
    │   ├── 10만+ 이미지
    │   ├── 수천 개 영상
    │   └── 검색 → 더블클릭으로 클립에 적용
    └── 개별 교체
        ├── 마음에 안 드는 클립만 선택 → AI 이미지 재생성
        └── 직접 이미지/영상 업로드로 교체
         │
         ▼
[5단계: 편집 및 보완]
    ├── 배경음악 추가 (200+ BGM, Mupat 소스)
    ├── 효과음 추가 (1000+ 효과음)
    ├── 전환효과(트랜지션) 적용
    ├── 애니메이션 효과 (등장/퇴장/강조/확대)
    └── 자막 스타일링 (폰트 100+, 색상, 배경, 위치)
         │
         ▼
[6단계: 내보내기]
    └── MP4 영상 / SRT 자막 / 텍스트 / 오디오 / 이미지
```

### 2.3 음성 분석 워크플로우 (기존 영상 편집)

```
[1단계: 영상 업로드]
    └── 영상 파일 불러오기
         │
         ▼
[2단계: 음성 인식 설정]
    ├── 발화 언어/사투리 정확히 지정
    ├── 음성이 선명하면 "음성 우선" 옵션 활성화
    ├── 배경 소음이 크면 노이즈 억제 먼저 적용
    └── 다중 오디오 트랙 시 음성 트랙 선택
         │
         ▼
[3단계: 자동 자막 생성]
    ├── AI가 음성을 분석하여 자막 자동 생성
    ├── 타이밍까지 자동으로 맞춤
    └── 클립(문장) 단위로 자동 분할
         │
         ▼
[4단계: 자막 편집]
    ├── 텍스트 수정 (오타/인식 오류)
    ├── 클립 나누기 (Enter 키) / 합치기 (Backspace 키)
    ├── 무음 구간 자동 삭제 (200~300ms 임계값)
    ├── 말버릇("음", "어" 등) 자동 정리
    └── 상세편집: 파형(waveform)을 보면서 타이밍 미세 조정
         │
         ▼
[5단계: 스타일링 및 내보내기]
    ├── 자막 서식 (글꼴, 크기, 색상, 배경)
    ├── 부분 서식 지정 (특정 글자만 다른 색/크기)
    └── SRT / MP4 내보내기
```

### 2.4 텍스트 수정 ↔ 음성 자동 갱신

Vrew의 핵심 장점 중 하나:
- TTS로 생성한 영상에서 **텍스트를 수정하면 AI 음성도 자동으로 재생성**됨
- 잦은 수정에도 빠르게 대응 가능
- 음성 속도/피치 변경 시에도 타이밍이 자동 재조정됨

## 섹션 3: 타임라인 UI 구조 상세

### 3.1 Vrew의 독특한 편집 패러다임

Vrew는 **전통적 타임라인 에디터가 아니다**. "문서 편집 방식"이 핵심이다.

#### 화면 4영역 구성

```
┌──────────────────────────────────────────────┐
│  [상단] 메뉴 바 + 도구 패널                      │
│  (파일, 편집, 삽입, 캡션, 효과 등)                │
├────────────────────┬─────────────────────────┤
│                    │                         │
│  [좌측]            │  [우측]                  │
│  미리보기 창        │  자막 편집기              │
│  (프리뷰 패널)      │  (텍스트 리스트)          │
│  - 편집 중인        │  - 클립별 자막 텍스트     │
│    영상 확인        │  - 직접 수정 가능         │
│  - 자막 프리뷰      │  - Enter: 클립 나누기    │
│  - 부분 서식 지정   │  - Backspace: 합치기     │
│                    │                         │
├────────────────────┴─────────────────────────┤
│  [하단] 타임라인 영역                            │
│  - 영상과 자막 타이밍 조절                        │
│  - 클립 블록 드래그로 시간 조정                    │
│  - 파형(waveform) 표시                          │
└──────────────────────────────────────────────┘
```

### 3.2 클립의 2층 구조

Vrew의 가장 기본적인 편집 단위는 **클립(Clip)**이다.

```
┌─────────────────────────────────┐
│  영상줄 (Video Row) ← 상단      │  ← 해당 구간의 이미지/영상
├─────────────────────────────────┤
│  자막줄 (Subtitle Row) ← 하단   │  ← 해당 구간의 자막 텍스트
└─────────────────────────────────┘
        ↕ 하나의 클립 단위
```

- **영상줄 (윗부분)**: 해당 시간 구간에 보여질 이미지 또는 영상
- **자막줄 (아랫부분)**: 해당 시간 구간의 자막 텍스트
- 클립의 길이 = TTS 음성 길이 또는 원본 영상의 해당 구간 길이

### 3.3 클립 편집 방식

| 동작 | 방법 | 설명 |
|------|------|------|
| 클립 나누기 | `Enter` 키 | 영상줄에 커서 → Enter → 2개로 분할 |
| 클립 합치기 | `Backspace` 키 | 앞/뒤 클립 커서 → Backspace → 1개로 병합 |
| 자동 클립 나누기 | [편집] 메뉴 | AI가 문맥+글자수 기준으로 자동 분할 |
| 이미지 교체 | 더블클릭 | 무료 에셋 검색 → 더블클릭으로 적용 |
| 이미지 복사 | `Ctrl+C/V` | 클립 간 이미지 복사 → [교체하기] 버튼 |
| 타이밍 조정 | 드래그 | 타임라인에서 클립 블록 양 끝을 드래그 |
| 상세편집 | 파형 보기 | 파형(waveform)을 보면서 세밀한 타이밍 조정 |

### 3.4 오디오 트랙과 비주얼 트랙의 관계

Vrew에서 오디오와 비주얼은 **클립 단위로 결합**되어 있다:

```
클립 1              클립 2              클립 3
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 이미지 A     │    │ 이미지 B     │    │ 이미지 C     │  ← 비주얼 트랙
├─────────────┤    ├─────────────┤    ├─────────────┤
│ "안녕하세요" │    │ "오늘은"     │    │ "감사합니다" │  ← 자막(텍스트)
├─────────────┤    ├─────────────┤    ├─────────────┤
│ TTS 음성 1   │    │ TTS 음성 2   │    │ TTS 음성 3   │  ← 오디오 트랙
└──┤ 0:00 ├──┘    └──┤ 0:03 ├──┘    └──┤ 0:06 ├──┘
   │ ~0:03│          │ ~0:06│          │ ~0:09│
```

- TTS 음성의 길이가 클립의 기본 듀레이션을 결정
- 이미지/영상은 해당 듀레이션에 맞춰 자동 조정
- 배경음악(BGM)은 별도 레이어로 전체 영상에 깔림
- 음량 밸런스: 음악 -20~-16 LUFS / TTS -14~-12 LUFS 권장

### 3.5 "문서 기반 편집"의 핵심 메커니즘

전통적 NLE(Non-Linear Editor)와의 비교:

| 측면 | Premiere/DaVinci | Vrew |
|------|------------------|------|
| 편집의 중심 | 타임라인 (시간축) | 텍스트 (문서) |
| 영상 자르기 | Razor 도구 → 타임라인에서 분할 | 텍스트 삭제 → 자동 제거 |
| 순서 변경 | 클립 드래그 | 텍스트 순서 변경 → 영상 자동 재배치 |
| 구간 삭제 | 구간 선택 → Delete | 텍스트 선택 → Delete |
| 타이밍 조정 | 클립 끝 드래그 | 텍스트 수정 → TTS 재생성 → 자동 조정 |
| 학습 곡선 | 높음 | 매우 낮음 (문서 편집 수준) |

## 섹션 4: AI 기능들

### 4.1 AI 이미지 생성

#### 4.1.1 AI 이미지 자동 삽입
- **진입**: [삽입] → [AI 이미지 자동삽입]
- **범위**: 선택 클립 또는 전체 클립
- **처리**: 한 번에 최대 10개 클립씩 배치 처리
- **매칭 로직**: 각 클립의 자막 텍스트 내용을 분석하여 어울리는 이미지 자동 생성
- **무료 비디오 포함 옵션**: 체크 시 일부는 무료 스톡 이미지/영상으로, 나머지는 AI 생성 이미지로 채움

#### 4.1.2 15가지 이미지 스타일
사실적인 사진부터 추상적인 아트까지 15가지 스타일 선택 가능:
- 실사 사진 스타일
- 애니메이션 스타일
- 다양한 색상 톤 커스텀
- (구체적인 15개 스타일 목록은 공식 문서에서 확인 필요)

#### 4.1.3 이미지 프롬프트 기능
- **위치**: [파일] → [새로 만들기] 시 또는 AI 이미지 자동 삽입 시
- **역할**: 전체 영상의 AI 이미지 분위기를 통일
- **예시**: "한복을 입은 사람들", "동아시아인", "미래적인 도시"
- 입력한 프롬프트가 모든 클립의 AI 이미지 생성에 반영됨

#### 4.1.4 개별 이미지 재생성/교체
- 마음에 들지 않는 이미지: 해당 클립 선택 → AI 이미지 재생성
- 직접 이미지 삽입: [삽입] → [이미지 · 비디오] → 파일 선택
- 무료 에셋 교체: 우측 무료 에셋 패널에서 검색 → 더블클릭 적용

### 4.2 AI 음성(TTS) 기능

| 항목 | 상세 |
|------|------|
| 음성 수 | 200+ (공식), 600+ (영문 페이지 기준) |
| 지원 언어 | 5개국어 (한/영/일/중/스페인어) |
| 음성 종류 | 남성, 여성, 어린이, 어르신, 나레이션, 대화형 |
| 속도 조절 | 자유 조절 (자연스러운 범위: 0.9~1.05) |
| 피치 조절 | ±1 반음 이내 자연스러움 |
| 내 목소리 AI | 자신의 목소리를 학습시켜 AI 클론 생성 |
| 강세 조정 | 핵심 단어 앞뒤에 쉼표 추가로 조절 |
| 자동 갱신 | 텍스트 수정 시 AI 음성도 자동 재생성 |

### 4.3 AI 대본 생성

- **주제만 입력** → AI가 완성된 스크립트 작성
- 톤, 스타일 등 추가 지시 가능
- "텍스트로 비디오 만들기" 모드에서 사용
- 대본 + 이미지 + 음성을 **한 문장만으로** 자동 생성 가능

### 4.4 AI 자막 관련 기능

| 기능 | 설명 |
|------|------|
| 자동 자막 생성 | 음성 인식 → 자막 자동 생성 + 타이밍 동기화 |
| 무음 구간 자동 삭제 | 200~300ms 임계값으로 무음 감지 → 자동 제거 |
| 말버릇 자동 정리 | "음", "어" 등 필러 워드 자동 감지 → 삭제 |
| 자동 클립 나누기 | AI가 문맥 + 글자수 기준으로 자동 분할 |
| 다국어 번역 | 60+ 언어 번역, 듀얼 자막 표시 지원 |
| 맞춤법 검사 | 자막 텍스트 맞춤법 자동 확인 |
| SRT 내보내기 | UTF-8(BOM) 형식으로 한글 깨짐 방지 |

### 4.5 AI 영상 효과

- **트랜지션 효과**: 줌 인/아웃, 글리치, 빠른 이동 등 클릭 한 번 적용
- **이미지 애니메이션**: 등장/퇴장, 강조, 확대(Ken Burns 유사) 효과
- **텍스트 애니메이션**: 등장/퇴장, 강조 효과
- **영상 확대/회전**: 줌 + 팬 기능
- **사용자 지정 애니메이션**: [효과] → [확대] → [사용자 지정]

### 4.6 VoyagerX의 기술 스택 (참고)

VoyagerX는 AWS 블로그에서 **ComfyUI 워크플로우 기반 AI 비디오 생성 파이프라인**을 구축했음을 공개했다.
(상세 기술은 비공개이나, ComfyUI + Stable Diffusion 계열 모델 사용을 시사)

## 섹션 5: Export/렌더링 플로우

### 5.1 내보내기 형식

| 형식 | 용도 | 비고 |
|------|------|------|
| **MP4 영상** | 최종 영상 파일 | 해상도/품질 선택 가능 |
| **SRT 자막** | 외부 자막 파일 | UTF-8(BOM) 권장 (한글 깨짐 방지) |
| **XML** | Premiere Pro 연동 | 자막 스타일 100% 반영 안됨 |
| **TXT** | 순수 텍스트 | 대본 추출용 |
| **오디오** | 음성 파일 | TTS 결과물 추출 |
| **이미지** | 스틸 이미지 | 썸네일 등 |
| **투명 배경 자막** | 오버레이용 | 다른 편집기에서 합성용 |

### 5.2 MP4 영상 내보내기 상세

```
[파일] → [영상으로 내보내기]
    │
    ├── 해상도 선택 (720p / 1080p / ...)
    ├── 화질 선택 (낮음 / 보통 / 높음 / 매우 높음)
    │   └── "높음 이상"이면 육안으로 구분 어려운 수준의 화질 유지
    ├── 하드웨어 가속 선택
    │   ├── ON: CPU보다 빠름, 파일 사이즈 약간 큼
    │   └── OFF: 더 안정적, 더 높은 압축률
    └── [내보내기] 클릭
        └── FFmpeg 기반 인코딩 수행
```

### 5.3 프레임레이트 및 타임코드

- 타임코드 드리프트 발생 시 → 프레임레이트 재확인 필요
- 첫 타임코드를 `00:00:00,000`으로 정렬 권장
- YouTube SRT 업로드: 언어별 개별 업로드 → 사용자 언어에 자동 매칭

### 5.4 내보내기 제한사항

- **웹 체험판**: 영상 내보내기 불가 (데스크톱 앱 필수)
- **무료 플랜**: 워터마크 포함 (유료 전환 또는 YouTube 표기로 제거 가능)
- **AI 비용**: 생성 시점에 크레딧 차감 (미저장 콘텐츠도 차감됨)

## 섹션 6: AntiGravity에 차용 가능한 패턴 정리

### 6.1 핵심 차용 패턴

#### Pattern 1: "텍스트 중심 타임라인" (Text-First Timeline)

**Vrew 방식**: 텍스트(자막)가 편집의 중심, 타임라인은 보조
**AntiGravity 적용**:

```
기존 구상:   스토리보드 → 이미지 생성 → 타임라인(시간축 중심)
개선 가능:   스토리보드 → 이미지 생성 → **대본 중심 타임라인**
```

- TimelinePage에서 씬별 대본 텍스트가 편집의 주축이 되게 함
- 각 텍스트 블록(클립)에 이미지/영상이 결합된 구조
- 텍스트 순서 변경 → 영상 순서 자동 변경

#### Pattern 2: "클립 = 텍스트 + 비주얼 + 오디오" 3-in-1 구조

**Vrew 방식**: 클립 하나가 영상줄 + 자막줄로 구성

**AntiGravity 적용**:

```typescript
interface TimelineClip {
  id: string;
  // 텍스트 레이어
  script: string;          // 씬 대본
  subtitle: string;        // 자막 텍스트
  // 비주얼 레이어
  imageUrl: string;        // AI 생성 이미지
  videoUrl?: string;       // AI 생성 영상 (있으면)
  animationEffect: 'none' | 'zoom-in' | 'zoom-out' | 'pan-left' | 'pan-right';
  // 오디오 레이어
  ttsAudioUrl?: string;    // TTS 음성 파일
  duration: number;        // 클립 길이 (TTS 기준 자동 계산)
  // 메타
  order: number;
  sceneId: string;
}
```

#### Pattern 3: "AI 이미지 자동 삽입" 패턴

**Vrew 방식**: 자막 텍스트 분석 → 15가지 스타일 중 선택 → 전체 클립에 AI 이미지 배치 생성

**AntiGravity 적용**:

```
IdeaPage(대본 작성)
    → StoryboardPage(씬 분할 + 스타일 선택)
        → GeneratePage(AI 이미지 배치 생성)  ← Vrew의 "AI 이미지 자동삽입"과 유사
            → TimelinePage(개별 교체/재생성 + TTS + 타이밍)
```

- "이미지 프롬프트" 개념 차용: 전체 영상의 통일된 스타일/분위기를 지정하는 글로벌 프롬프트
- 개별 클립별 재생성 기능 (마음에 안 드는 것만 교체)
- 무료 에셋 대체 옵션 (크레딧 절약)

#### Pattern 4: "TTS → 타이밍 자동 계산" 패턴

**Vrew 방식**: TTS 음성 길이가 클립 듀레이션을 결정

**AntiGravity 적용**:

```
[Animate(타임라인) 단계]
1. 씬별 대본 텍스트 확정
2. AI TTS로 음성 생성 → 각 클립의 duration 자동 계산
3. AI 이미지/영상을 해당 duration에 맞춤
4. 이미지에 Ken Burns 효과(확대/패닝) 자동 적용으로 "정지 이미지의 영상화"
5. 전환효과(트랜지션) 클립 사이에 자동 삽입
6. 배경음악 별도 레이어로 깔기
7. 최종 렌더링 → MP4
```

#### Pattern 5: "문서 편집식 UX" (Document-like Editing)

**Vrew 방식**: 텍스트 삭제 = 영상 삭제, 텍스트 이동 = 영상 이동

**AntiGravity 적용**:
- TimelinePage의 좌측을 "대본 리스트 뷰"로 구성
- 대본 텍스트를 편집하면 → TTS 재생성 → 타이밍 자동 조정
- 드래그로 씬 순서 변경 → 전체 타임라인 자동 재배치
- 텍스트 삭제 → 해당 씬의 이미지/영상/TTS 함께 삭제

### 6.2 차용하지 않을 것 (안티패턴)

| Vrew 방식 | 이유 | AntiGravity 대안 |
|-----------|------|------------------|
| 단순 슬라이드쇼 수준의 영상 | 시네마틱 퀄리티 목표와 불일치 | AI 영상(Runway/Kling) 우선, 이미지는 Ken Burns 효과 |
| 워드(단어) 단위 클립 | 시네마틱 영상에선 씬 단위가 적합 | 씬(문장~문단) 단위 클립 유지 |
| 자동 무음 삭제 | 시네마틱에선 "여백"도 연출 | 수동 타이밍 조절 우선 |
| 스톡 이미지/영상 자동 채움 | 통일된 스타일 유지 어려움 | AI 생성 이미지 + 씨드 매칭으로 일관성 유지 |

### 6.4 구현 우선순위 제안

| 순위 | 기능 | Vrew 참고 | 난이도 |
|------|------|----------|--------|
| 1 | 씬 리스트 (클립 = 이미지 + 대본) | 클립 2층 구조 | 중 |
| 2 | TTS 음성 생성 + 자동 타이밍 | TTS → duration 자동 계산 | 높음 (API 연동) |
| 3 | 이미지 Ken Burns 효과 | 확대/팬 애니메이션 | 중 |
| 4 | 씬 순서 드래그 변경 | 텍스트 순서 변경 → 영상 재배치 | 중 |
| 5 | BGM 트랙 | 배경음악 별도 레이어 | 중 |
| 6 | 전환효과 (트랜지션) | 클릭 한 번 적용 | 중 |
| 7 | MP4 내보내기 | FFmpeg 기반 렌더링 | 높음 (서버 필요) |
| 8 | 대본 수정 → TTS 자동 갱신 | 텍스트 수정 → 음성 재생성 | 높음 |

### 6.5 핵심 인사이트 요약

1. **텍스트가 왕이다**: Vrew의 성공 비결은 영상 편집을 "텍스트 편집"으로 추상화한 것. AntiGravity도 시네마틱 영상 제작을 "대본 편집"으로 추상화해야 한다.

2. **TTS가 타이밍을 결정한다**: 수동으로 타이밍을 맞추는 것은 초보자에게 너무 어렵다. TTS 음성 길이가 자동으로 각 씬의 듀레이션을 결정하는 패턴이 핵심.

3. **AI 이미지 배치 생성 + 개별 교체**: 전체를 한 번에 생성하되, 마음에 안 드는 것만 개별 교체하는 UX가 효율적. "이미지 프롬프트"로 전체 분위기를 통일하는 것도 좋은 패턴.

4. **클립 단위의 결합**: 이미지 + 텍스트 + 오디오를 하나의 클립으로 묶는 구조가 초보자에게 직관적. 전문가용 멀티트랙 타임라인은 불필요.

5. **렌더링은 서버 사이드**: Vrew는 데스크톱 앱이라 로컬 FFmpeg를 쓰지만, AntiGravity는 웹 앱이므로 서버 사이드 렌더링(또는 클라우드 렌더링)이 필요. 이 부분이 가장 큰 기술적 도전.
