# Supabase 실연결 - 사용자 작업 가이드

> 이 가이드를 따라하면 약 10~15분이면 완료됩니다.

---

## 📋 해야 할 것 요약 (3단계)

1. Supabase 프로젝트 만들기 (3분)
2. DB 테이블 만들기 (2분)
3. Google 로그인 설정 (10분)

---

## STEP 1: Supabase 프로젝트 만들기

1. https://supabase.com 접속 → 회원가입/로그인 (GitHub 계정으로 가능)
2. **"New Project"** 클릭
3. 입력:
   - Project name: `antigravity`
   - Database Password: 아무거나 (메모해두세요)
   - Region: **Northeast Asia (Seoul)** ← 꼭 서울!
4. **"Create new project"** 클릭 → 1~2분 기다리기

### ✅ 복사할 값 2개
프로젝트 생성 후:
1. 좌측 ⚙️ **Settings** → **API** 탭
2. 아래 2개를 메모장에 복사:
   - **Project URL**: `https://xxxxx.supabase.co`
   - **anon public** 키: `eyJhbGci...` (긴 문자열)

---

## STEP 2: DB 테이블 만들기

1. 좌측 메뉴 **SQL Editor** 클릭
2. **"New Query"** 클릭
3. 프로젝트 폴더의 `supabase/migrations/001_initial_schema.sql` 파일 내용을 **전체 복사**하여 붙여넣기
4. **"Run"** 클릭
5. **"Success"** 메시지 확인

---

## STEP 3: Google 로그인 설정

### 3-1. Supabase에서 준비
1. 좌측 **Authentication** → **Providers** 탭
2. **Google** 클릭 → **Enable** 토글 ON
3. 아래에 표시된 **Callback URL** 복사 (`https://xxxxx.supabase.co/auth/v1/callback`)
4. (아직 Save 누르지 마세요!)

### 3-2. Google Cloud Console
1. https://console.cloud.google.com 접속
2. 상단에서 프로젝트 선택 (없으면 **"새 프로젝트"** → 이름: AntiGravity)
3. 좌측 메뉴 **"API 및 서비스"** → **"사용자 인증 정보"**
4. (처음이면) **"OAuth 동의 화면"** 설정:
   - User Type: **External**
   - 앱 이름: `AntiGravity`
   - 이메일 입력
   - 나머지 빈칸 OK → 저장
5. 다시 **"사용자 인증 정보"** → **"사용자 인증 정보 만들기"** → **"OAuth 클라이언트 ID"**
6. 입력:
   - 유형: **웹 애플리케이션**
   - 이름: `AntiGravity Web`
   - 승인된 JavaScript 원본: `http://localhost:5173`
   - 승인된 리디렉션 URI: 3-1에서 복사한 Callback URL 붙여넣기
7. **"만들기"** 클릭
8. 나온 **클라이언트 ID**와 **클라이언트 보안 비밀번호** 복사

### 3-3. Supabase에 입력
1. Supabase Authentication → Providers → Google으로 돌아가기
2. **Client ID** 붙여넣기
3. **Client Secret** 붙여넣기
4. **Save** 클릭

### 3-4. Redirect URL 설정
1. Authentication → **URL Configuration**
2. **Site URL**: `http://localhost:5173`
3. **Redirect URLs**: `http://localhost:5173/**` 추가
4. Save

---

## 📌 완료 후 Claude에게 전달할 것

아래 2개 값만 Claude에게 알려주시면 됩니다:

```
VITE_SUPABASE_URL=https://여기에-복사한-URL
VITE_SUPABASE_ANON_KEY=여기에-복사한-anon-key
```

> ⚠️ 이 값들은 `.env` 파일에 들어갑니다. 절대 공개 채팅에 올리지 마세요.
> (anon key는 클라이언트용이라 노출돼도 RLS가 보호하긴 하지만, 습관적으로 비공개 유지!)
