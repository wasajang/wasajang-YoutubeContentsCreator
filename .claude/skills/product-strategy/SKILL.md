# AntiGravity 프로덕트 전략 가이드

## 프로덕트 핵심 정체성

**한 줄:** YouTube 크리에이터가 AI로 시네마틱 영상을 제작하고, 스타일을 공유/판매하는 플랫폼.

### 우리가 아닌 것 (Anti-Positioning)
- 범용 AI 영상 도구가 아님 (→ "YouTube 크리에이터 전용")
- 단순 AI 생성기가 아님 (→ "시네마틱 워크플로우")
- 기술자용 도구가 아님 (→ "비개발자도 쓸 수 있는 쉬운 UX")

### 핵심 차별점 (Moat)
1. **3진입점 워크플로우** — 대본/스타일/Cast 어디서든 시작
2. **Cast AI 스튜디오** — 일관된 캐릭터 Seed 관리
3. **UGC 마켓** — 프리셋 공유/판매 (크리에이터 경제)
4. **BYOK 하이브리드** — 파워유저 비용 절감

## 의사결정 프레임워크

### 기능 우선순위 판단 기준 (RICE)
- **Reach:** 몇 %의 사용자에게 영향?
- **Impact:** 사용자 가치 얼마나 큰가? (1~3)
- **Confidence:** 확신도 (높/중/낮)
- **Effort:** 개발 공수 (S/M/L/XL)

**RICE 점수 = (Reach x Impact x Confidence) / Effort**

### Go/No-Go 체크리스트
- [ ] VISION.md의 방향과 일치하는가?
- [ ] MVP 범위 내인가? (아니면 "나중에")
- [ ] 타겟 사용자(YouTube 크리에이터)에게 직접적 가치가 있는가?
- [ ] 기존 아키텍처에서 구현 가능한가?
- [ ] 수익 모델에 기여하는가?

## 수익 모델 핵심

### 2계층 크레딧
- **플랫폼 이용료:** 항상 차감 (BYOK여도)
- **AI API 비용:** BYOK 시 면제

### 가격 정책 원칙
1. 무료 플랜으로 진입 장벽 낮추기
2. Pro 전환 시 "10배 이상 가치" 느끼게
3. BYOK로 파워유저 확보 (저비용 → 고빈도 사용)
4. UGC 판매 수수료로 크리에이터 경제 형성

## 로드맵 우선순위 (현재 기준)

```
[완료] 001-Supabase 연결
[완료] 002-UX 플로우 개선
[다음] 003-AI 실 연동         ← MVP 핵심 (실제 AI 동작)
[이후] 004-결제 시스템         ← 수익화
[이후] 005-UGC 마켓           ← 성장
[이후] 006-배포               ← 런칭
```
